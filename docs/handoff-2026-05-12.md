## 2. 資料與業務邏輯現況

### 2.1 月結狀態

目前 `snapshot.status` 有兩種：

- `draft`
- `final`

現在實際行為差異：

- `draft`：可在月結列表刪除
- `final`：月結列表不提供刪除

目前仍然 `可以編輯 final 月結`。  
也就是說，現在的 `final` 比較像「狀態標記」，不是嚴格的唯讀鎖定。

如果未來希望：

- `定稿後不可編輯`
- `只能複製定稿`
- `定稿後需另開修正版`

這些都還需要另外實作。

### 2.2 匯率策略

目前 USD 匯率採用 `月份層級`：

- 每個 `snapshot` 有一個 `usdFxRate`
- 當月所有 `USD` 項目統一使用這個匯率
- `snapshotEntry.fxRate` 仍會保存建立當下的值，作為歷史快照

這代表：

- 使用者不需要逐筆輸入匯率
- 過往月份的匯率會保留，不會被主檔或後續月份覆蓋

### 2.3 Demo 資料

目前資料庫已建立兩年份 demo 月結資料：

- 範圍約為 `2024-06` 到 `2026-05`
- 共 `24` 個月份
- 含資產、負債、TWD / USD、備註與波動金額

建立腳本：

- [scripts/seed-demo-snapshots.ts](/Users/huatsanhuang/Programs/finance/scripts/seed-demo-snapshots.ts)

用途：

- 用來看列表變長時的 UI
- 看 Dashboard 趨勢圖
- 看報表頁密度
- 看月結編輯頁在多筆資料下的排版

## 5. 目前已知風險 / 尚未完成

### 5.1 B 版尚未完整 QA

雖然程式已完成，但 `B` 版目前沒有做完所有頁面的人工視覺 QA。

建議接手時優先逐頁確認：

3. 單月月結編輯
4. 報表頁
5. 分類管理
6. 項目管理

### 5.2 定稿尚未唯讀

如前面所述，目前 `final` 不是唯讀。

如果未來使用者期待：

- 定稿後不可再動
- 輸入框 disabled
- 不可新增 / 移除 / 改金額

需要另外補商業規則與 UI 鎖定。

### 5.3 in-app browser 連線狀態不穩

在這輪開發中，曾出現：

- in-app browser session 消失
- pane 不在 automation session 內
- 無 active browser pane

這不是專案頁面本身的問題，比較像工具執行環境的 session 狀態。  
接手時若要視覺驗證，可能需要：

- 手動刷新 in-app browser
- 重新開頁
- 重新接回 browser-use session

### 5.4 Next.js dev cache 可能壞掉

曾出現嚴重樣式失效，原因是：

- HTML 指向的 `/_next/static/css/...` 資源回傳 `Not Found`
- 導致頁面退回未套樣式狀態

當時的處理方式：

1. 停掉 dev server
2. 將 `.next` 改名備份
3. 重啟 dev server

備份目錄：

- `.next-cache-20260512-155123`

並且已經在 ESLint ignore 加入：

- `.next-cache-*/**`

相關檔案：

- [eslint.config.mjs](/Users/huatsanhuang/Programs/finance/eslint.config.mjs)
